## TUFF-OS について
The Ultimate Fortress Foundation OS (TUFF-OS) は、物理層での絶対的なデータ主権を提供するセキュリティ特化型 OS です。

## 配布
- **バージョン**: 1.1.0 (LiveUSB インストーラ版)
- **ターゲット**: x86_64 UEFI Secure Boot 準拠
- **最新イメージ**: `latest/TUFF-OS-latest.iso`
- **イメージ形式**: LiveUSB への書き込みと UEFI 起動を前提とした ISO
- **イメージ生成**: 標準リリースツールで生成し、手作業での組み立てや編集は行いません。

## ISO の扱い方
1. 同じコミットの配布物をまとめて取得してください。
   - `latest/TUFF-OS-latest.iso`
   - `latest/TUFF-OS-latest.iso.sigv1`
   - `latest/update-metadata.json`
2. `latest/TUFF-OS-latest.iso` は展開せず、そのまま USB メモリへ raw 書き込みしてください。ISO 内のファイルを手でコピーしても起動用メディアにはなりません。
3. 書き込み例:
   - Linux: `sudo dd if=latest/TUFF-OS-latest.iso of=/dev/sdX bs=4M status=progress oflag=sync`
   - Windows: Rufus または balenaEtcher を使い、イメージ書き込みで USB を作成
4. 作成した USB を UEFI モードで起動し、インストーラの案内に従ってください。
5. ミラー配布や保管時は、ISO 単体ではなく `.sigv1` と `update-metadata.json` も同じ組で保持してください。

## 差分更新の適用方法
- 差分パッケージは `deltas/*.tpatch` に配置されます。
- 通常ユーザーは `.tpatch` を手で展開・再生成しないでください。インストール済み TUFF-OS 上で `tuffutl TUFFOS update` を実行します。
- 更新処理は `latest/update-metadata.json` を読み、対応する `deltas/v<from>_to_v<to>.tpatch` を選択し、`patch_sig` を検証してから差分適用し、最後に再構築結果を `full_iso_sig` と照合します。
- 対応する差分がない場合は、フル ISO を使う経路へ自動でフォールバックします。
- 署名検証に失敗した場合は即中止し、このリポジトリから同一コミットの配布物一式を再取得してください。別コミットの ISO / `.sigv1` / `update-metadata.json` を混在させないでください。
