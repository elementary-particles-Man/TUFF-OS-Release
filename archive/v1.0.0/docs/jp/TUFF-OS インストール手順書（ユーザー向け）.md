# TUFF-OS インストール手順書（ユーザー向け）

**最終更新**: 2026年3月22日  
**対象**: Windows 11 / Linux ホスト上でTUFF-OSを導入したい方  
**所要時間目安**: 約60〜120分（ストレージ初期化時間による）

### 重要警告（必ず最初に読んでください）

- **全データ消失の危険性**があります。  
  TUFF-OSインストーラは**対象ディスクの全データを物理的に消去**します。  
  → 重要なデータは事前に**別ドライブへ完全バックアップ**してください。
- 物理HDDを複数接続する場合は**メーカー・容量がバラバラでも問題ありません**が、  
  **最低3台以上**（推奨5台）を用意すると3N冗長の信頼性が飛躍的に向上します。
- インストール中は**電源を絶対に切らない**でください。  
  電断時は自動ロールバック機能が働くよう設計されていますが、念のためUPS推奨。

### 準備するもの

1. **ホストマシン**（Windows 11 または Linux）
2. **USBメモリ**（8GB以上、FAT32フォーマット済み）
3. **物理HDD**（3台以上推奨、SATA接続）
4. **インターネット接続**（初回ビルド時のみ必要）
5. **管理者権限**（sudo / 管理者として実行）

### 手順

#### Step 1: インストーラの入手
1. 公式リポジトリ（または配布元）から最新のインストーラをダウンロード  
   - ファイル名例: `tuff-installer-latest.exe`（Windows） / `tuff-installer`（Linux）
2. **SHA256ハッシュ**を確認し、改ざんがないことを検証（推奨）

#### Step 2: バックアップとディスク準備
1. 重要なデータは**別のドライブへ完全コピー**  
2. 対象となるHDDを接続（最低3台、推奨5台）  
3. **ディスク管理**（Windows）または `lsblk`（Linux）でデバイス名を確認  
   例: Windows → `\\.\PhysicalDrive2`、`\\.\PhysicalDrive3` など  
   Linux → `/dev/sdb`、`/dev/sdc` など  
   → **間違ったドライブを指定するとデータが消えます**。必ずダブルチェック。

#### Step 3: インストーラの起動
**Windowsの場合**
1. 管理者権限で `tuff-installer-latest.exe` を実行
2. 「同意する」をクリックして進む

**Linuxの場合**
```bash
sudo chmod +x tuff-installer
sudo ./tuff-installer
```

#### Step 4: インストールウィザードの進行

1. **言語選択** → 日本語を選択（推奨）
2. **対象ディスクの選択**  
   - システムインストール先（SSD推奨）：1台選択  
   - データJBODプール（HDD）：3台以上選択（Ctrl+クリックで複数選択）
3. **警告ダイアログ**  
   → 「選択した全ドライブのデータが永久に消去されます。よろしいですか？」  
   → **「はい、理解しました」** を2回クリック（二重確認）
4. **初期設定**  
   - 管理者パスワード（12桁以上推奨）  
   - TUFFkey.jsonの保存先（USBメモリ推奨）
5. **インストール開始**  
   - 所要時間：約20〜60分（HDD台数と容量による）

#### Step 5: インストール完了後の確認

1. 再起動後、**UEFIブートメニュー**から「TUFF-OS」を選択  
2. 初回ブート時にGenesis初期化が自動実行されます（数分）
3. デスクトップに**tuffutl**アイコンが表示されたら成功
4. コマンドプロンプト / ターミナルで以下を実行し、状態を確認

```bash
tuffutl sys status
```

期待される出力例：
```
Genesis: Valid (3N majority OK)
UserAuthDB: Initialized (3 replicas detected)
Isolation: Inactive
Disk Pool: Healthy (5/5 drives)
```

#### Step 6: 初回ユーザー作成（必須）

```bash
tuffutl sys user add yourname --password "StrongPass123!"
```

- 初回ログイン時に**パスワード変更が強制**されます
- 変更後、**tuffutl sys login** でセッション開始

#### トラブルシューティング（よくある失敗と対処）

| 症状                                   | 原因の可能性                     | 対処法                                      |
|----------------------------------------|----------------------------------|---------------------------------------------|
| インストーラが途中で停止               | 電源断 / ディスク不良            | ログ確認後、再インストール（自動ロールバック） |
| `tuffutl sys status` でGenesis Invalid | 3Nのうち2台以上破損             | 破損ディスクを交換 → `tuffutl sys fsck`     |
| 上位OSからHDDが見えない               | 正常（TUFF-OSが隠蔽）            | `tuffutl sys login` 後にマウント           |
| 10Gbps攻撃でCPUが上がる               | Vulkanパススルー未設定           | QEMU/KVMで `-device vfio-pci` 設定確認     |

---

**インストール完了後の推奨初期設定**

1. `tuffutl nw blacklist add` で危険IPを登録
2. `tuffutl sys user add admin2 --admin` で予備管理者作成
3. `tuffutl fs commit` を定期的に実行（手動 or スケジュール）
4. 重要なフォルダにTag（例: "社外秘"）を付与

これでTUFF-OSの基本運用が開始できます。

**安全第一**  
TUFF-OSは物理層でデータを守りますが、**インストール前のバックアップはユーザーの責任**です。  
万が一のデータ消失リスクを理解した上でご利用ください。
