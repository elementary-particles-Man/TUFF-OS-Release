**TUFF-OS Isolationモード 詳細説明**

Isolationモードは、TUFF-OSの**最終防衛ライン**であり、  
「不正アクセスや異常を検知した瞬間に、システムを物理的・論理的に完全遮断する」機能です。  
他のセキュリティ機構（Deception、3N冗長、ネットワーク防衛層など）が突破された場合、または多層防御の最後の砦として作動します。

### 1. トリガー条件（発動する瞬間）

Isolationモードは以下のいずれかで即座に発動します（優先順位が高い順）：

1. **偽トークン / なりすまし検知**（最も頻発するトリガー）  
   - HMAC-SHA3-256署名付きセッショントークンの検証に3回連続失敗  
   - 同一ユーザーで重複ログイン試行（セッションID衝突）×3回

2. **ネットワーク防衛層の異常検知**  
   - 10Gbps級SYN Flood + L7ペイロード攻撃が継続  
   - IDPI（AI Probe）で「悪意ある応答パターン」検知（例：exfiltrate secrets系プロンプト）

3. **Genesis / 3N Consensus Failure**  
   - 3N多数決で2台以上不一致（物理改ざん疑い）

4. **手動トリガー**（管理者用）  
   - `tuffutl sys isolation trigger`

5. **タイムアウト / 異常終了**  
   - GPUオフロード応答が5ms以上タイムアウト  
   - CommitPending状態で電断検知（MQ未処理）

### 2. 発動時の挙動（何が起きるか）

Isolationモードに入ると、以下の処理が**即時に順次実行**されます：

1. **全セッション即時破棄**  
   - ZRAM上のセッション鍵・TagGroupMaskをAVX2/AVX-512で一括Zeroize  
   - すべてのアクティブセッションを強制Logout

2. **全物理I/O完全遮断**  
   - 上位OSからのRead → 永久ランダムノイズ返却（ChaCha20 + LBA位相）  
   - Write → STATUS_SUCCESSを偽装しつつ実際はSilent Drop  
   - 新規ファイル作成 / ディレクトリ操作 → ENOENT（存在しない）偽装

3. **ネットワーク全遮断**  
   - ネットワーク防衛ルールが「全パケットDrop」へ切り替え  
   - aiserverlist / 許可リストも一時無効化

4. **Handoffフラグ永続化**  
   - TuffHandoffBlock V1 に「Isolation Persistent」フラグをセット  
   - 次回再ブート後も自動でIsolation状態を復元

5. **witness.logへの最終記録**  
   - トリガー原因 + タイムスタンプ + ML-DSA量子耐性署名で記録  
   - ログ自体も即座に暗号化・ハッシュチェーン化

### 3. Isolation解除方法（唯一の復帰経路）

解除は**極めて厳格**に制限されています。

1. **管理者専用Recovery PIN**  
   - インストール時に生成・USBメモリに暗号保存されたPIN  
   - コマンド：`tuffutl sys isolation recover --pin <12桁PIN>`

2. **物理的再インストール**（最終手段）  
   - USBインストーラから再実行 → Genesis再初期化  
   - 旧データはすべて破棄（Fail-Closed）

**通常ユーザー**は解除できません。  
**PINを紛失した場合**は、再インストール（全データ消去）しか方法がありません。

### 4. 解除後の挙動

- セッションはゼロから再構築（再ログイン必須）
- ネットワークはKAIROの初期ポリシーに戻る（blacklistは保持）
- witness.logに「Isolation recovered」記録（PQC署名付き）

### 5. 実際の運用例（シナリオ別）

| シナリオ | トリガー | 結果 | 所要時間 |
|----------|----------|------|----------|
| 偽トークン攻撃 | 3回連続失敗 | 全I/O遮断 + Zeroize | 即時 |
| 大規模DDoS | KAIRO GPU検知 | Silent Drop継続 + CPU 0% | 継続中 |
| ディスク物理改ざん | 3N不一致 | ブート停止 + Isolation | 起動時即時 |
| ランサムウェア侵入 | J世代検知 | 即時ロールバック + Isolation | 数ms |

### まとめ：Isolationモードの哲学

- 「**疑わしきは即座に隔離**」  
- 「**痕跡を残さず、存在を秘匿**」  
- 「**再ブート後も防衛を継続**」

これにより、TUFF-OSはAIエージェントや国家級攻撃者に対しても、  
**物理的・永続的な主権**を守り続けます。
